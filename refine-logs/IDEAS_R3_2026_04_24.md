# Round 3 Ideas — Targeting ≥7/10 Novelty

## R3-A: EPR-X — Falsifiability-Gated Elicited Predicate Reward

**Claim:** A self-elicited target-shape reward that *only fires when the prediction passes a cheap, schema-grounded falsifier*, with adversarial-target controls baked into the loss — not as an ablation.

**Mechanism.** Before SQL generation, the model emits a structured target shape `T = (n_cols, col_types, agg_present, group_keys_count)` conditioned on (question, schema). We construct three counterfactual targets in the same minibatch:
- `T_rand`: random shape from schema-feasible distribution
- `T_corrupt`: T with one field flipped
- `T_oracle`: extracted from gold SQL (training-only, dropped at deployment)

Reward: `r_EPR = sigmoid(s(T) - s(T_rand)) * 1[falsifier(T)=valid]`, where `falsifier` checks (a) col-count ≤ schema width, (b) types compatible with question class (e.g., "how many" → INT agg), (c) group keys ⊆ mentioned entities. Add a *contrastive* term during warm-up: `s(T_oracle) > s(T_corrupt) > s(T_rand)` margin loss. At RL time only `r_EPR` is used.

**Critique addressed.** R2 reviewer on EPR: *"deadliest attack is rewarding agreement with hallucinations… only helps if predicted target is informative."* Contrastive ranking + falsifier gate make informativeness a *training-time identifiability constraint*, not an empirical hope.

**Closest prior.** Self-Consistency Decoding (2203.11171), Self-Refine (2303.17651), Plan-and-Solve (2305.04091), VerifyToM (2410.16429). None combine adversarial shape contrasts with falsifier gate inside RL reward.

**Anticipated attack.** "Falsifier is a hand-coded heuristic — gains are from falsifier alone." Mitigation: ablate falsifier-only baseline; expect near-zero gain.

**24GB pilot cost.** Qwen2.5-3B + LoRA, 8 rollouts × 4 prompts/step, ~14h for 800 GRPO steps on BIRD-mini. Shape extractor regex over gold AST: free.

**Decisive ablation.** Replace `T` predictions with `T_oracle` at RL time. If EPR-X with predicted T recovers ≥80% of oracle-T performance → mechanism real. <30% → fake.

## R3-B: ITE — Interventional Turn Effect Estimator

**Claim:** A per-turn credit assignment estimator using *paraphrase-preserving interventions* and *execution-state transition ATEs* — first causal turn-credit method for Text-to-SQL agents.

**Mechanism.** For trajectory τ = (s₀, a₁, s₁, ..., a_T, s_T), at turn t form `do(a_t := ã_t)` where `ã_t` ~ paraphrase distribution `p(·|intent(a_t))` — implemented by re-prompting the same policy with "rewrite this reasoning step preserving its decision; alter wording only." Collect K=4 paraphrases, filter via NLI entailment classifier for intent-preservation, match by token-entropy bucket and action-type (think/sql/observe). Counterfactual rollouts greedy-completed from s_t. Turn-effect: `ITE_t = E[Φ(s_T) | a_t] − E[Φ(s_T) | ã_t]`, where Φ = *execution-state delta* (rows-returned, type-match, schema-coverage), not terminal success. GRPO advantage: `A_t = ITE_t + λ · R_terminal`.

**Critique addressed.** R2 on CATS: *"pivot to actual interventional estimator with paraphrase-matched counterfactuals, ATE on downstream execution-state transitions (not just terminal success), entropy-matched controls."* Every demanded element built into estimator definition.

**Closest prior.** Counterfactual Story Reasoning (1909.04076), CausalLM attribution (2305.00050), Token-Level ATE for RLHF (CausalRLHF line ~2503), Step-DPO (2406.18629). None operate at *agentic turn level* with execution-state Φ as outcome.

**Anticipated attack.** "Paraphrase distribution is policy-defined → intervention support overlaps with observational distribution → just IS-weighted advantage." Defense: NLI-filtered paraphrases reduce KL but preserve action-decision; show overlap-vs-effect curve to demonstrate strict sub-distribution intervention.

**24GB pilot cost.** K=4 counterfactual rollouts × T≈6 turns × 200 prompts ≈ 5k extra rollouts/epoch. NLI via DeBERTa-v3-base on CPU. ~22h/epoch; budget 2 epochs.

**Decisive ablation.** Replace ITE_t with random per-turn advantage of equal variance. If ITE matches random → noise. If ITE > random by >5pt EX → real causal signal.

## R3-C (WILDCARD): SKE-RL — Skeleton-Equivalence-Class Reinforcement

**Claim:** GRPO over *discrete SQL skeleton equivalence classes*, not over surface SQL strings. Execution feedback credits the right structural decision; literal filling absorbs nuisance variance.

**Mechanism.** Define skeleton extractor `σ: SQL → z` mapping any SQL to its α-renamed AST template (predicates abstracted to `pred_i(col, ?)`, literals to typed holes). Policy emits `(z, fill)` jointly. Reward `R(z, fill) = EX(SQL(z, fill)) + β · max_{fill'} EX(SQL(z, fill'))` — skeleton gets *partial credit if any feasible filling works*, computed by sampling K=8 fillings via small filler head. GRPO advantage computed at **skeleton level**: rollouts in same equivalence class `[z]` share advantage. Add skeleton-entropy bonus to maintain plan diversity.

**Critique pre-empted.** Avoids reward shaping (skeleton credit is execution-grounded), avoids IS/off-policy (no replay), avoids distillation (no SFT injection), avoids info-gain bonuses (no entropy-of-success), avoids difficulty curriculum.

**Closest prior.** SkelSQL/RESDSQL (2302.05965), DIN-SQL (2304.11015), Plan-of-SQLs (2403.11102), MIPS discrete-latent RL (2106.06134), Latent Plans for RL (2402.16928). **None apply skeleton-equivalence-class advantage estimation in agentic GRPO.**

**Anticipated attack.** "Skeleton extraction is inductive bias — optimal SQL might not fit your AST family." Defense: report skeleton-coverage on BIRD/Spider gold (>95% expected); fall back to surface GRPO for uncovered cases.

**24GB pilot cost.** Skeleton extractor: AST walk, free. K=8 fillings via greedy decode of 0.5B filler head OR same 3B with constrained decoding. ~18h for 1k GRPO steps on BIRD-mini.

**Decisive ablation.** Set K=1 (no within-skeleton search) AND disable skeleton-class advantage sharing → reduce to literal-SQL GRPO with same compute. If SKE-RL ≥ literal-GRPO + 4pt EX → skeleton-equivalence structure is the cause.
