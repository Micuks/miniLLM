# Round 2 Ideas — Targeting ≥6/10 Novelty

## Idea R2-1: CATS — Counterfactual Action-Turn Surgery

**Mechanism.** For each successful trajectory τ = (s₀,a₀,...,sT,aT), identify "pivotal" turns by **action ablation**: replay the trajectory but resample only turn t from π_old at temperature T̃, holding all prior turns fixed via teacher-forced KV cache. If success rate drops >δ when t is resampled, t is pivotal. Apply a **turn-weighted advantage** Â_t = A · (1 + λ·pivotality_t), normalized within group. No gold SQL touched; pivotality comes from execution outcomes only.

**Dodges R1 critiques.** Not rejection sampling. Not replay. No gold leakage — pivotality from binary execution downstream.

**Closest prior.** COMA (arXiv:1705.08926) for fully-observable MARL. No close LLM-agent analog.

**Pilot cost.** k=4 ablation rollouts × per turn surgery ≈ 5× base. Tight on 24GB with rollout batch 4.

**Anticipated attack.** "Pivotality = leverage of high-entropy turns; you've rediscovered entropy weighting." Defense: empirical decorrelation between pivotality and entropy.

## Idea R2-2: TIPS — Turn-factored Importance with Per-Step Stop-grad

**Mechanism.** Decompose π(τ|q) = Π_t π(a_t | s_t) where s_t includes deterministic SQL execution observations (env-deterministic). Standard GRPO uses sequence-level ratio r(τ) = Π_t π_θ(a_t)/π_old(a_t) with compounding variance over T turns. Replace with **per-turn ratio** r_t = π_θ(a_t|s_t)/π_old(a_t|s_t) and apply PPO clipping per turn, then sum: L = Σ_t min(r_t·Â, clip(r_t)·Â). Stop-gradient on s_t (prior tokens + env returns) so no inter-turn credit smearing.

**Dodges R1 critiques.** Sequence vs turn-level IS is a real algorithmic distinction. Not GRM-adjacent. Exploits env-determinism explicitly.

**Closest prior.** ArCHer (arXiv:2402.19446) hierarchical decomposition with critic. Group Sequence Policy Optimization debates exist but turn-factoring with env-determinism exploit is not standard.

**Pilot cost.** Zero extra rollouts. ~5% extra forward cost for per-turn logprob bookkeeping.

**Anticipated attack.** "Variance-reduction claim needs empirical demonstration on grad-norm trajectories, not theory."

## Idea R2-3: EPR — Execution-Prefix Process Reward

**Mechanism.** During rollout, after each SQL exec turn, extract structural features of the result: (column count, row count bucket {0,1,few,many}, presence of NULL columns, type signature). Define **prefix-monotonicity reward**: r_t = +β if turn t's exec result is structurally closer to a *self-elicited* target signature obtained by asking the model "describe expected result shape" in a separate frozen-model query at trajectory start. Cumulative process reward Σ r_t added to terminal reward.

**Dodges R1 critiques.** Target signature comes from policy's pre-rollout prediction, not gold. No DB-state leakage. Distinct from PORG (no edit distance, no ordering).

**Closest prior.** PRM math literature (arXiv:2305.20050, Lightman et al.) uses human/gold-derived step labels. Self-elicited shape for SQL is, AFAIK, novel.

**Pilot cost.** 1 extra forward pass per question for shape prediction. ~10% overhead.

**Anticipated attack.** "Self-elicited shape is hallucination; you reward agreement with hallucinations." Real concern.

## Idea R2-4: SAGA — Schema-Action Grounding Auxiliary

**Mechanism.** Parse each emitted SQL with a lightweight AST walker; extract referenced (table, column) pairs. Compare against the schema in the prompt — *not* against gold SQL. Reward signal: r_schema = (# valid refs) / (# total refs) − γ·(# hallucinated refs). Rewards grounding *in the prompt's schema* (input, not label). Apply as dense per-turn reward additive to terminal correctness.

**Dodges R1 critiques.** Schema = input, not label. No gold answer involvement.

**Closest prior.** PICARD (arXiv:2109.05093) constrained decoding for schema validity at decode time. RAT-SQL (arXiv:1911.04942) uses gold links. Pure-schema reward without gold links uncommon.

**Pilot cost.** AST parse microseconds; negligible.

**Anticipated attack.** "Schema validity is necessary but not sufficient. Model can write `SELECT * FROM correct_table` and be valid but wrong."

## Idea R2-5: CD-GRPO — Constrained-Decoding GRPO with KL-to-Unconstrained

**Mechanism.** Generate rollouts under SQL-grammar constrained decoder (Outlines/SynCode-style, arXiv:2403.01632). Eliminates format-hack failure modes by construction. Importance-correct policy gradient since π_constrained ≠ π_θ: ratio w(τ) = π_θ(τ)/π_constrained(τ) for off-policy correction; KL(π_θ || π_constrained_unconditional) regularizer prevents drift into constraint-masked degenerate regions.

**Dodges R1 critiques.** Removes failure mode rather than gating against it. Operates at decoding/sampling level, not reward shaping.

**Closest prior.** GAD (arXiv:2406.13631) trains under grammar constraints for code; not multi-turn ReAct + GRPO.

**Pilot cost.** Constrained decoding 1.3-2× slower per token. ~2× rollout cost. Tight on 24GB with batch 2.

**Anticipated attack.** "Constraint masks symptom — underlying reasoning is bad. Will show 100% format compliance with no execution gain."

## Idea R2-6: PIB — Posterior Information Bonus

**Mechanism.** At turn t, before emitting next action, sample K=4 candidate next-turn completions from π_old. Score each by terminal-reachability proxy: complete each via greedy continuation, score by execution. Define **posterior entropy** H_t = entropy over (success/fail) outcomes of K continuations. Bonus reward: b_t = max(0, H_{t-1} − H_t) — rewards turns that *collapse uncertainty* about success. Bayesian-experimental-design-flavored exploration bonus.

**Dodges R1 critiques.** Not entropy regularization (which adds H to reward); rewards *change in posterior entropy* — directional, information-gain quantity. Per-rollout, not per-prompt → can't be sampling-freq artifact (ATE critique).

**Closest prior.** VIME (arXiv:1605.09674) for classical RL exploration via posterior info gain. No LLM-agent analog. Self-Consistency (arXiv:2203.11171) measures agreement, not info gain.

**Pilot cost.** K=2 lookahead × depth=2 ≈ 6× base. Tight on 24GB; may need rollout batch 1.

**Anticipated attack.** "K=2 lookaheads have huge H variance; bonus is noise."
